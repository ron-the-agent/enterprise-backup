Three bugs fixed, each with a different root cause:


Fix 1 — BackupStats move constructor (lines 495–516)
std::atomic deletes its copy and move constructors, so any struct containing atomics can't be returned by value either. The fix is an explicit move constructor that manually load()s each atomic's value into the new object. This is the standard pattern for movable structs with atomic members.


Fix 2 — Ternary iterator (line 1310)
fs::recursive_directory_iterator and fs::directory_iterator are completely different types — the compiler can't unify them in a single auto ternary expression. The fix splits it into two separate if/else loops, each with the right iterator type. The duplicated loop body could be extracted into a lambda if you want to avoid repetition.


Fix 3 — unique_ptr type mismatch (line 1409)
std::unique_ptr<MmapCopyStrategy> and std::unique_ptr<BufferedCopyStrategy> are unrelated types even if both classes share a common base — you can't assign one to the other. 
The fix declares the strategy as std::unique_ptr<CopyStrategy> (the base class), then assigns the correct derived type based on the mode. 
This is the standard polymorphism pattern with smart pointers.
